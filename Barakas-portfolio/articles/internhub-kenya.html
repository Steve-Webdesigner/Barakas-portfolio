Introduction

Every developer dreams of building something that makes a real difference. For me, that project is InternHub Kenya—a platform designed to connect Kenyan students and fresh graduates with verified internship opportunities, attachments, graduate trainee programs, and entry-level positions. This article takes you behind the scenes of how I planned, designed, built, and deployed the platform.

The Idea

The idea for InternHub Kenya came from a problem I observed around me. Many talented students and graduates in Kenya struggle to find internship opportunities. Job boards are either too general, outdated, or require paid subscriptions. Meanwhile, companies looking for interns lack a centralized platform to post opportunities and reach the right candidates.

I realized that by building a dedicated platform specifically for the Kenyan market, I could bridge this gap. The platform would focus on verified opportunities, provide a clean and intuitive interface, and be accessible to anyone with an internet connection—completely free for students.

Planning

Before writing a single line of code, I spent weeks planning the project. I started by defining the core features:

  • Browse internships, attachments, and graduate trainee programs
  • Filter by category, location, and field of study
  • Detailed opportunity pages with application instructions
  • Responsive design for mobile users
  • Admin dashboard for posting and managing opportunities
  • Search functionality

I created wireframes on paper first, then moved to Figma for high-fidelity mockups. The planning phase helped me visualize the user flow, identify potential challenges, and make design decisions before coding began.

Technologies Used

I chose technologies that I was comfortable with and that would allow me to build a fast, reliable, and maintainable platform:

  • HTML5 — Semantic markup for structure and SEO
  • CSS3 — Custom styles with CSS Grid, Flexbox, and custom properties
  • Vanilla JavaScript — DOM manipulation, filtering, search, and interactivity
  • Vercel — Deployment and hosting
  • Git & GitHub — Version control and collaboration

I deliberately chose vanilla JavaScript over frameworks like React to keep the project lightweight and to strengthen my understanding of core JavaScript concepts. This decision also makes the platform fast to load and easy to maintain.

Problems Solved

Throughout development, I encountered several significant challenges:

Data Management. Initially, I hardcoded internship listings in the JavaScript file. As the number of listings grew, this became unmanageable. I transitioned to using a JSON-based data structure, which allowed me to separate content from code and make updates easier.

Search and Filtering. Implementing a fast and accurate search and filter system was more complex than I anticipated. I needed to filter by multiple criteria—category, location, type—simultaneously. I built a filtering function that iterates through the data array and returns only matching results, with debounced input for better performance.

Mobile Responsiveness. Many students access the internet primarily through mobile devices. Ensuring a seamless experience on small screens required careful CSS planning, touch-friendly buttons, and optimized image sizes.

SEO Optimization. Since the platform needs to be discoverable, I invested time in proper meta tags, semantic HTML structure, descriptive alt texts, and optimized page titles.

💡 Solution: I created a reusable data management pattern using JavaScript modules. Each opportunity is a data object stored in an array. Functions like filterOpportunities() and renderCards() handle display logic. This separation of concerns makes the code cleaner and more maintainable.

Lessons Learned

Building InternHub Kenya taught me invaluable lessons:

  • Start simple, then iterate. I launched with core features first and added enhancements based on user feedback.
  • User testing is crucial. I asked friends and colleagues to test the platform. Their feedback revealed issues I had never considered.
  • Documentation saves time. I started documenting my code late in the project and regretted it. Documentation helps you and others understand the codebase.
  • Deploy early, deploy often. Pushing to production early gave me motivation and revealed deployment-specific issues sooner.

Future Improvements

InternHub Kenya is not finished—it's constantly evolving. Here are my planned improvements:

  • User authentication and account creation
  • Application tracking system for students
  • Employer dashboard for posting and managing listings
  • Email notifications for new opportunities
  • Integration with WhatsApp for instant sharing
  • Blog section with career advice articles

Conclusion

Building InternHub Kenya has been my most ambitious project to date. It taught me the importance of planning, the value of user-centered design, and the satisfaction of creating something that serves a real community need. If you're a student developer looking for a project to build, I encourage you to identify a problem in your community and build a solution. The experience you gain will be invaluable.
